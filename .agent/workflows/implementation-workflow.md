# Implementation Workflow — TokDig

## ⚠️ ATURAN WAJIB (Harus diikuti SETIAP SAAT)

### SEBELUM mulai implementasi apapun:

1. **Baca Project Overview**
   - Buka dan baca artifact `project_overview.md` di brain directory
   - Pahami status terkini project, apa yang sudah selesai, dan apa yang belum

2. **Baca Implementation Plan**
   - Buka dan baca artifact `implementation_plan.md` di brain directory
   - Identifikasi step/task mana yang akan dikerjakan
   - Pastikan task sebelumnya sudah selesai (jangan skip urutan)

3. **Cek file dokumentasi project** (jika relevan)
   - `fitur_web_digital_product.md` — untuk detail fitur
   - `system_architecture.md` — untuk arsitektur sistem
   - `database_schema.md` — untuk struktur database
   - `feature_roadmap.md` — untuk roadmap phase
   - `tech-stack.md` — untuk tech stack

### SESUDAH selesai implementasi:

4. **Update Implementation Plan**
   - Tandai task yang sudah selesai ✅
   - Update status di checklist
   - Catat perubahan atau penyesuaian yang dilakukan
   - Tambahkan task baru jika ditemukan kebutuhan tambahan

5. **Update Project Overview**
   - Update status terkini project
   - Update informasi yang berubah (jika ada perubahan arsitektur, schema, dll)

### ATURAN TAMBAHAN:

- **JANGAN** mulai coding tanpa baca plan terlebih dahulu
- **JANGAN** skip step di implementation plan tanpa alasan jelas
- **SELALU** sinkronkan artifact dengan kondisi aktual project
- **SELALU** informasikan ke user progress terkini setelah update plan
